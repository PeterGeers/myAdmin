1. System Overview

The myAdmin Banking Processor is a sophisticated transaction processing system designed to automate the import and categorization of bank transactions from multiple financial institutions. The system processes CSV and TSV files, applies intelligent pattern matching for account classification, and maintains comprehensive duplicate detection.

Key Features

  • Multi-Bank Support: Rabobank CSV, Revolut TSV, Generic CSV formats
  • Intelligent Pattern Matching: Automatic debit/credit account assignment
  • Duplicate Prevention: Sequence-based duplicate detection per IBAN
  • Test/Production Modes: Safe testing environment with separate database tables
  • Real-time Validation: Field validation and error highlighting
  • Historical Pattern Learning: Learns from previous transaction categorizations

5. Pattern Matching System

Historical Pattern Learning

Pattern Storage Logic

System learns from historical transactions to automatically assign debit/credit accounts:

# Query historical patterns patterns = db.query(""" SELECT referenceNumber, debet, credit, administration FROM mutaties WHERE administration = %s AND referenceNumber IS NOT NULL GROUP BY referenceNumber, debet, credit """, administration) # Build pattern groups for pattern in patterns: if pattern.debet and pattern.debet < '1300': debet_patterns[pattern.referenceNumber] = { 'debet': pattern.debet, 'credit': pattern.credit }

Pattern Application

Automatic Account Assignment

# Apply patterns to new transactions for transaction in transactions: description = transaction.TransactionDescription # If Credit is empty, try debet patterns if not transaction.Credit: for reference, accounts in debet_patterns.items(): if re.search(reference, description, re.IGNORECASE): transaction.ReferenceNumber = reference transaction.Credit = accounts.credit break # If Debet is empty, try credit patterns elif not transaction.Debet: for reference, accounts in credit_patterns.items(): if re.search(reference, description, re.IGNORECASE): transaction.ReferenceNumber = reference transaction.Debet = accounts.debet break
Pattern Scope: Patterns are administration-specific and only apply to accounts below 1300 (operational accounts, not bank accounts)

6. Duplicate Detection

Sequence-Based Detection

IBAN-Specific Sequence Tracking

Each bank account (IBAN) maintains its own sequence numbering:

# Check existing sequences for specific IBAN existing_sequences = db.query(""" SELECT DISTINCT Ref2 FROM mutaties WHERE Ref1 = %s AND Ref2 IS NOT NULL """, iban) # Filter out duplicates new_transactions = [ t for t in transactions if t.Ref2 not in existing_sequences ]

Pattern Extraction Query

10. Troubleshooting

Common File Processing Issues

Encoding Problems

  • Rabobank CSV: Must be Latin-1 encoded (not UTF-8)
  • Revolut TSV: Must be UTF-8 encoded
  • Symptoms: Garbled characters, parsing errors
  • Solution: Check file encoding in text editor

Column Mismatch Errors

  • Cause: Bank changed export format
  • Symptoms: Wrong data in fields, missing descriptions
  • Solution: Update column mappings in processor code

Database Connection Issues

  • Check .env file: Verify database credentials
  • Test connection: Use database client to verify connectivity
  • Table permissions: Ensure user has INSERT/SELECT rights

Pattern Matching Problems

  • No patterns applied: Check if historical data exists for administration
  • Wrong accounts assigned: Review pattern history and clean up duplicates
  • Missing reference numbers: Manually assign and save to build patterns
